Quick reference for everything below.

Core editing

  • Enter / F2 → edit
  • Enter → new sibling below (while editing)
  • Shift + Enter → split at the cursor — text after the cursor moves into a new sibling below, text before it stays put
  • Ctrl/Cmd + Enter → new child — works whether you're editing a node or just have one selected
  • Type → start editing

Navigation

  • Click → select
  • Shift + Click → range
  • Ctrl/Cmd + Click → multi-select
  • Ctrl/Cmd + A → select all

Structure

  • Right-click a node → a row of icon buttons for the most common actions; defaults are Zoom in, Note, Tags, Rewrite, Decision Log, Delete. Click More for move, sort, rewrite document, and everything else. Customise which buttons appear via Settings → Right-click menu — you can also reset to defaults there. Zoom in is also in the toolbar Fold group and via Ctrl/Cmd + .; zoom out via Ctrl/Cmd + , or clicking the same toolbar button again
  • Backlinks — while editing a node, type @ to mention another node by name; a dropdown filters as you type, pick with mouse or arrow keys + Enter. The reference renders as a clickable link that jumps to the target. Open the Note panel on any node to see which nodes link to it. Deleting a referenced node removes its links from all other nodes automatically
  • Tags — select a node and click the tag icon (toolbar Insert group) or right-click → Tags to open the tag popover. Existing document tags appear as chips to toggle; type a new name and press Enter to add. Tags render as #chips on the node. Click a chip in the tree to filter to that tag; click again or × to clear. Multi-select: toolbar button applies to all selected nodes
  • Checkboxes — click the ☑ toolbar button (Insert group) to toggle a node as a checkbox. Or type [ ] / [x] at the start of a node while editing — it auto-converts on commit. Click the box to check/uncheck. Parent checkbox nodes with checkbox children show a 2/5 progress badge; checking all children auto-checks the parent. Checked nodes fade. Multi-select supported
  • ⤴ → insert above
  • ＋ → add child
  • Tab / Shift + Tab → indent / outdent
  • ⧉ → duplicate node / subtree
  • Alt + ↑ / ↓ → move
  • Drag a row → drop above/below to reorder, or onto the middle of another row to nest as its child
  • Right-click a node with children → More → Sort children (A → Z, Z → A, or by depth — shallowest subtree first)
  • Extras in the toolbar → sort all top-level nodes the same three ways

Tagging

  • Select a node and click the tag icon (toolbar Insert group) or right-click → Tags to open the tag popover. Existing document tags appear as chips to toggle; type a new name and press Enter to add. Remove a tag from a node by toggling its chip off in the popover
  • Tags render as #chips directly on the node row. Click a chip in the tree to filter the entire tree to that tag; click the chip again or the × to clear the filter
  • Multi-select: select several nodes, then click the tag icon — the popover applies to all selected nodes at once. Tags already on all selected nodes appear toggled on; tags on only some appear in an indeterminate state
  • AI Suggest tags (AI toolbar group) reads the node text and the document's existing tags, then applies relevant ones directly — prefers reusing tags already in the document before inventing new ones

@ Mentions & Backlinks

  • While editing a node, type @ to mention another node by name; a dropdown filters as you type — pick with mouse or arrow keys + Enter. The reference is saved as [[Node name]] and renders as a clickable link that jumps to the target node
  • Open the Note panel on any node to see its Backlinks section — every node that @mentions it appears there as a clickable entry. If no backlinks exist the section is hidden
  • Deleting a referenced node automatically removes its [[mention]] links from all other nodes
  • Turning the feature off (Settings → Features → Backlinks) suppresses the @ dropdown and hides the Backlinks section in the note panel. Existing [[mentions]] in node text still render as clickable links regardless — they navigate on click even when the feature is off

Focus (zoom into a branch)

  • Right-click a node → Zoom in (or Ctrl/Cmd + ., or the zoom icon in the toolbar Fold group) → only that node and its descendants are shown. A breadcrumb bar appears above the editor showing the path back to the root; click any crumb to zoom out to that level
  • Exit focus with Ctrl/Cmd + ,, by clicking the same toolbar button again, or by clicking the root crumb in the breadcrumb bar
  • Focus state is per-tab and is not saved — closing and reopening a document always starts at the full tree
  • Turning the feature off (Settings → Features → Focus) hides the zoom-in toolbar button and right-click action, and exits any active focus session immediately. Ctrl/Cmd + . also stops working while the feature is off

Formatting

  • Ctrl/Cmd + B → bold
  • Ctrl/Cmd + I → italic
  • Ctrl/Cmd + U → underline
  • Ctrl/Cmd + Shift + S → strike
  • Ctrl/Cmd + Shift + H → highlight (caret next to it picks a color)
  • Ctrl/Cmd + Shift + F → text color (caret next to it picks a color)

Semantics

  • [Text] → [Section] — blue label
  • (Text) → (note) — muted italic
  • !Text → !alert — red warning
  • `Text` → `code` — inline code
  • >Text → quoted or attributed content — starts the node with >; hides the > in view mode. Use for direct quotes, attributed statements, or cited requirements
  • @Node name → Node name — backlink to another node; type @ while editing to pick from a list. Open the Note panel on the target node to see all backlinks pointing to it
  • [ ] task or [x] task → auto-converts to a checkbox node on commit
  • Tag icon in toolbar or right-click → Tags → tag popover (toggle existing tags or add new ones)

AI Rewrite

  • ✦ button in the toolbar → rewrites the selected node; with multiple nodes selected, rewrites all of them in one batch; with text highlighted inside an editing node, rewrites just that selection
  • Right-click a node → Rewrite node is in the quick icon row (Rewrite selection, if multiple are selected); click More → Rewrite document to rewrite every node in the tree
  • button next to Rewrite, or Ctrl/Cmd + Shift + O → Generate outline (AI): describe a topic and it inserts an AI-generated nested outline into the current document — into a blank doc, or after the current selection; also available as "Generate with AI" on the blank-document screen. Uses the same provider/model/key as Rewrite (Settings → AI Rewrite), but its own fixed instructions, not the customizable rewrite prompt
  • button next to Outline → Suggest icon: picks a fitting emoji for the selected node's label and adds it as a prefix (e.g. "🛒 Webshop"). With multiple nodes selected, applies one straight away to each, picked from a keyword list first and an exact match against icons already used elsewhere in your documents/templates, falling to AI only when neither matches. For a single node, if more than one good option turns up, a small popover appears above it to pick from; right-click → More → Suggest icons for all nodes runs it over the whole tree (asks to confirm first). Since the icon is just leading text, removing or changing it is normal editing — double-click the node and edit it like any other text
  • Batches larger than 30 nodes are sent in sequential chunks automatically, with a status toast showing progress (e.g. "Rewriting nodes 1–30…")
  • Expand node (AI) — select a single node and click the expand button in the AI toolbar group to break it into child nodes. Useful for dense labels like "Distribution model (OPML, localStorage, multi-tab sync)" → a proper subtree. AI generates 3–8 children; result is undoable
  • Suggest tags (AI) — select a node and click the tag-suggest button; AI reads the node text and the document's existing tags, then applies relevant ones directly. Prefers reusing tags already in the document before inventing new ones
  • Summarise selection into parent (AI) — select 2 or more nodes (Shift-click or Ctrl/Cmd-click), then click the summarise button; AI writes a concise parent label and inserts it above the selection, with the selected nodes indented underneath. The bottom-up outlining pattern — build the detail first, name the group after
  • Settings → AI Rewrite → choose Gemini (free), Groq (free, fast), or Claude API, add your own API key, and customize the rewrite prompt ("Reset to default" next to the prompt box restores just its wording; "Reset AI settings" at the bottom of the section restores provider, model, and prompt together and clears the saved key). All AI features — Rewrite, Expand, Tag suggestion, Summarise, Generate outline, Suggest icon — share the same provider and key

Utilities

  • Ctrl/Cmd + S → save now (kept in-app — doesn't trigger the browser's "Save Page As")
  • Print button in the header (next to Import ▾ / Export ▾) → prints the document with all UI chrome hidden; choose "Save as PDF" in the browser's print dialog to get a clean PDF. Checked nodes and tag chips render correctly in print; the sidebar, toolbar, and status bar are all hidden automatically
  • Export ▾ is grouped into Copy / Document / Data / AI sections. Everything in it respects Settings → Export & print → "Skip folded sections" the same way — folded branches are included or skipped consistently across Markdown, Word, OPML, and the items below, not just whichever one you happen to use
  • Diagram ✦ (Export ▾ → AI) — copies a prompt to your clipboard that turns the current outline into a styled draw.io flowchart when pasted into Claude or any other AI assistant. Lines ending in "?" are flagged as decision points so the result gets diamonds/branches instead of plain boxes; the prompt also pins exact colors and styling to cut down on manual touch-ups afterward. If clipboard access is blocked, a dialog shows the prompt in a text box to copy manually instead
  • Save AI Diagram (Export ▾ → AI) — paste the AI's reply to the Diagram ✦ prompt here to turn it into a real .drawio file. Code fences and any stray sentences the AI adds before/after the XML are stripped automatically; if what you pasted isn't actually valid draw.io XML, it tells you why instead of saving a broken file
  • Ctrl/Cmd + K → open and focus the header search bar (to hide it, use the toggle in Settings → Header); it searches settings, help, document and template names, and now node text across all documents too — results group as "In documents" with a snippet, and clicking one jumps straight to that node
  • Ctrl/Cmd + Shift + D / Insert Date/Time in toolbar → timestamp
  • Copy Tree as Image → PNG
  • Ctrl/Cmd + Shift + C → copy selection
  • Clear → reset
  • Shortcuts that double as OS/browser bindings can occasionally be intercepted before they reach the page — if one doesn't fire, the matching button or menu item always works

Maximize / Zen mode

  • ⤢ floating button bottom-right of the editor, or Esc to exit → hides the sidebar, tab strip, and status bar so only the tree (and toolbar, if it's on) remains
  • A second floating button sits next to it — a shortcut for Settings → Toolbar → "Show toolbar," so it works the same way whether you're in zen mode or not. Click it, or Ctrl/Cmd + Shift + T, to show or hide the toolbar; the choice applies everywhere and is remembered next time
  • If you'd rather zen mode hide the toolbar by default, turn "Show toolbar" off before maximizing — everything it holds is still reachable by keyboard shortcut (see the other sections on this page) or by right-clicking a node for the icon row and "More" menu
  • The Pad panel has its own floating toggle next to the maximize button, since it stays open across zen mode

Preview

  • 👁 floating button bottom-right of the editor (next to zen/Pad/toolbar) → opens a read-only, presentation-style render of the current document. Nothing here can be edited, and closing it never changes your actual document
  • Headings carry depth 0–1; deeper levels become plain indented bullets with no tree-line connectors, so it reads like a document rather than code
  • Click ▾/▸ → fold/unfold a branch, same as the editor — but this fold state is local to Preview and separate from the editor's own collapsed sections. Collapse all / Expand all buttons sit in the header
  • Folding a branch that contains a note or a decision log doesn't hide it silently — a small icon next to the +N badge tells you it's in there; click either to expand
  • Notes render as their real rich content (images, lists, links) inside a card, not the small dot from the editor
  • Decision logs become a card with a "Decision Log" header — Status always shows there as a colored badge (color is guessed from the word itself: approved/accepted/implemented/resolved/done/confirmed → green, rejected/declined/blocked/abandoned/cancelled → red, hold/pending/review → orange, anything else → gray). The rest of the answered fields appear as a one-line summary below, with their labels kept neutral on purpose — Status is the only thing meant to carry color in this card. Which fields appear is configurable in Settings → Preview → "Decision card fields" (Context + Decision by default; this only affects how Preview summarizes a log, not what the Decision Log feature itself inserts)
  • Right-click anywhere in Preview → Laser pointer, for pointing at things while presenting without your real cursor in the way. Turns off automatically when you close Preview
  • Export buttons in the header (Markdown / Word) are the same export as everywhere else in Sakura — Preview just gives you a one-click path to them while you're already looking at the rendered result

Large trees

  • Click ▾/▸ → fold / unfold subtree
  • ← / → → collapse / expand selected
  • Ctrl/Cmd + F → search this tree; Ctrl/Cmd + H → search and replace (Replace / Replace all, undoable)
  • Ctrl/Cmd + Shift + [ → collapse all
  • Ctrl/Cmd + Shift + ] / ▼ in toolbar → expand all
  • ▶ in toolbar → collapse to a set depth (configurable in Settings)

Documents & folders (sidebar)

  • button in sidebar header → new folder; button → new document (unfiled); or Ctrl/Cmd + Alt + N from anywhere
  • Hover a folder → ＋ creates a document already filed there; ✎ renames it; deletes it (subfolders move up a level, documents move to Unfiled — Undo appears briefly after deleting)
  • Right-click a folder → new document or new subfolder here, rename, or delete
  • Right-click a document → rename, duplicate, move to a folder, or move to Trash
  • Click ✎ / ⧉ / icons on a document row → rename / duplicate / move to Trash; the folder-icon button moves it via a dropdown list
  • Click a document's or folder's name → switch to it / expand it
  • Folders can be nested: drag a folder onto another folder's top or bottom edge to reorder it as a sibling there, or onto the middle to nest it inside
  • Drag a document onto a folder (or "Unfiled") to file it there; drag it onto another document to reorder within that folder/Unfiled
  • / buttons by "Documents" → collapse all / expand all folders (only shown once a folder exists)
  • Ctrl/Cmd-click a document to multi-select; Shift-click selects a range. A bar appears to move or trash the whole selection at once
  • Deleting a document (or a selection of them) — or a template, including a built-in one — moves it to Trash, a collapsible section at the very bottom of the sidebar, below Templates — out of the way since it's rarely needed, same as most apps put it. It's not gone. Each item there shows when it was trashed, with buttons to restore it or delete it forever; documents and templates sit in the same list, told apart by their icon and a small "Template" label. A toast with "Undo" also appears right after trashing, for a quick reversal without opening Trash at all. Settings → Trash controls how long items sit there before being purged automatically (default 7 days; "Forever" turns off auto-purge so only "Empty Trash" removes anything) — one setting, shared by both. Header search (Ctrl/Cmd + K) skips Trash by default — turn on Settings → Trash → "Include Trash in search" to have it surface matches there too, clearly marked "In Trash"; clicking one reveals it in the Trash section rather than opening it directly, since restoring is a deliberate choice. Settings → Templates → "Restore built-in templates" also pulls a trashed built-in back out of Trash, not just refreshes its content
  • Searching the sidebar keeps folders visible (showing only matching documents and subfolders, force-expanded) but disables drag-and-drop while a search is active; it only matches document/template names — for text inside documents, use the header search (Ctrl/Cmd + K) instead
  • The sidebar's filter box is hidden by default since the header search now covers the same ground; bring it back via the icon next to "Documents", or Settings → Sidebar → Filter box
  • Drag the thin strip at the sidebar's right edge to resize it
  • Version history (right-click a document → "Version history", or "More" → "Version history" for the one currently open) keeps automatic checkpoints of a document's past content — not full version control, just lightweight snapshots. A version is captured right before it would otherwise be overwritten, but only once every ~10 minutes of active editing at most, and only if something actually changed, so it won't flood with near-duplicates from every autosave tick. Capped at the last 20 per document, oldest dropped first. "Save a version now" in that panel adds one immediately, useful right before a big rewrite. Restoring an older version is itself non-destructive — whatever was there gets saved as a version first, so restoring is never a one-way trip
  • If the same document is open as the active one in two tabs or windows of this browser at once (easy to do by accident — duplicating a tab, browser session restore, opening the file twice), a banner appears the moment the other one saves a change, since otherwise whichever tab saves last would silently win with no warning. "Reload latest" discards anything unsaved here and pulls in the other tab's version; dismissing it just clears the banner — your edits here are kept either way, and the other tab gets the same warning in reverse once you save. Documents open here only in the background (not the one on screen) refresh silently instead, since there's nothing unsaved there to protect. This only applies within one browser — a completely different browser, or a different device, never shares storage and can't collide

Templates (sidebar)

  • Click a template's name in the Templates section → load it into the current document (replaces current content; Undo restores it)
  • Default → loads your custom default template, if one is set
  • ⋯ button → Save current as template, or Set current as default
  • button next to ⋯ → new template folder, for grouping templates (single level; templates can be moved between folders via the move-to-folder button or right-click, but folders themselves can't be nested or reordered)
  • / buttons by "Templates" → collapse all / expand all template folders (only shown once a template folder exists)
  • Ships with a built-in pack — Meeting Notes, 1:1 Notes, Daily Standup, Weekly Status, Decision Record, Project Kickoff, Retrospective — grouped in a "Built-in" folder. The folder and the templates in it can be renamed and moved like any other, but are deleted like any other — deletion goes to Trash and is reversible. Use the ↺ restore button in the Templates sidebar header to restore any deleted or renamed built-in templates at any time. Icons are grouped by document shape, not unique per template: calendar for anything written on a recurring cadence, checkmark for the decision record, flag for one-off project milestones
  • The "Templates ▾" button in the header toolbar opens the same template list as a quick-access dropdown, grouped by folder exactly like the sidebar (icons included) — useful when the sidebar itself is hidden, e.g. in zen mode. It's read-mostly there: click a template to load it, plus rename/move-to-folder/delete per row, but folders themselves are only created from the sidebar
  • A line that's exactly "Date:", "Author:", or "Date · Author" in any template (built-in or your own) gets auto-filled the moment the template loads — today's date, and the document's current author if one is set (see Settings → Editing → "Default author" to pre-fill that automatically too). The merged "Date · Author" form fills in with no labels, just the values
  • Built-in templates use one color convention throughout, worth keeping if you duplicate and extend one: purple = neutral section header, orange = action/decision, red = risk/blocker, green = positive
  • Want a template in a new doc instead of overwriting? Create a new document first, then load the template

Tabs

  • Click a document in the sidebar, or the + at the end of the tab strip → opens it as a tab. Switching tabs keeps each one's undo history, scroll position, and selection independent — it doesn't reset like a plain document switch would
  • Double-click a tab → rename it
  • ✕ on a tab, or Ctrl/Cmd + Alt + W → closes the active tab only; the document itself isn't deleted and stays in the sidebar. Closing the last tab leaves the editor empty rather than reopening another document
  • Drag a tab left or right to reorder the strip
  • ▾ at the right of the tab strip → search all open tabs by name; ↑ / ↓ + Enter to jump, or click a result
  • Settings → General → "Reopen tabs on launch" controls whether your open tabs come back next time, or you start with just one

Backing up your data

  • Everything is stored locally in the browser, scoped to this exact file. Opening a different copy or a newer version of this file starts with empty storage — that's a browser limitation, not a bug
  • Settings → Data → Auto-backup to file → keeps a live backup written to a file on disk as you work (Chrome/Edge only — Safari and Firefox don't support this yet). Browsers revoke write permission to that file across a reload or restart as a deliberate security measure, so this can need reconnecting — a chip in the status bar flags it when that happens, and clicking it (or "Reconnect" in Settings) restores it in one click
  • Settings → Data → Local safety copy → a second copy kept automatically in a separate browser storage area; Restore brings it back if the main storage gets cleared. Not a substitute for Export — it's still inside the same browser
  • Settings → Data → Backup / Restore → still the way to move data between files, browsers, or computers, or to take a one-off manual backup; the sidebar footer has Backup/Restore shortcuts too, so you don't need to open Settings just for that. The header's Export ▾ and Import ▾ buttons are for the current document only (Markdown, Word, OPML, etc.) — Backup/Restore cover all your data
  • Recommended: export before switching to a new version of this file, then import right after opening it
  • Settings → Data → Reset to default settings → restores appearance/behavior preferences only (theme, toolbar, right-click menu, etc.); your documents and AI settings aren't touched

Note panel

  • Every node can hold a rich-text note — open it via the toolbar ( Note button), right-click → More → Note, or Ctrl/Cmd + Shift + N; clicking the chat-bubble dot that appears on a node with a note also opens it directly
  • The note panel is a floating, draggable window — drag it by its header to reposition it anywhere; click to expand to full-screen and click again to restore
  • Toolbar: B I U S for bold / italic / underline / strikethrough; bullet list and numbered list; insert or edit a link, with its own display-text field separate from the URL; table icon inserts a 2×2 table; AI Rewrite rewrites the whole note or just selected text; the summarise button reads the node's child subtree and appends a prose summary to the note
  • Tab → indents a list item (when inside a list) or inserts spaces in plain text; Shift + Tab → outdent a list item
  • Paste images directly into the note — they embed inline and are stored with the note. Click a pasted image to select it, then drag the handle at its corner to resize; double-click it to reset back to its original size
  • Tables: drag the thin border handle between columns to resize column widths
  • Ctrl/Cmd + Enter or Esc → close and save; "Clear note" removes it entirely (undoable)
  • A chat-bubble dot on a node row signals it has a note — click it to open that note without changing the selection
  • Backlinks — if other nodes reference the current node via @mention, a Backlinks section appears at the bottom of the note panel. Click any entry to jump to that referencing node
  • Timestamps — the note panel also shows when the node was created and last modified, on a single line at the bottom. Existing nodes show the time they were first loaded after this feature was added; new nodes and any node whose text is edited going forward will have accurate timestamps

Decision Log

  • Inserts a structured record under the selected node: a header line (timestamp + optional author) with child prompts for Context, Decision, Rationale, Alternatives, Impact, and Status
  • Insert via the toolbar's Decision Log button (in the Insert group), or right-click a node → More → Decision log
  • Settings → Decision Log → "Ask who decided" prompts for a name each time (on by default); the timestamp is always added regardless. "Highlight field labels" colors the Context/Decision/Rationale/etc. labels in the editor so they stand out from the rest of the text — it doesn't extend to the Decision Log card in Preview, which keeps its labels neutral on purpose so the colored Status badge is the only thing carrying color in that card
  • Settings → Decision Log → "Fields to include" lets you turn off any of the six prompts you don't use — e.g. skip Alternatives, Impact, and Status if you only ever fill in Context, Decision, and Rationale. The header itself always gets added; this only controls which child prompts come with it. Changes apply to logs you insert afterward, not ones already in a document
  • A small checkmark icon appears next to the "+N hidden" badge on a folded node when something inside its collapsed subtree contains a decision log, so you know there's one there before expanding
  • Export menu → Decision Log → Excel (.xlsx) pulls every decision log across the current document into one spreadsheet (timestamp, author, parent node, and whichever fields were filled in) — fields you've turned off, or left blank, just export as empty cells

Pad

  • A docked panel for whole-document notes — separate from per-node Notes above. One Pad per document, not tied to any single node
  • Open it with the small panel icon docked at the bottom-right of the editor, next to the maximize button, or Ctrl/Cmd + Shift + P; the same shortcut and button close it again. It stays open if the editor is maximized, and that floating button is how you open or close it while maximized too — handy if Settings → Toolbar → "Show toolbar" is off and the regular toolbar isn't around to use instead
  • Drag the thin strip on its left edge to resize it; width is remembered, as is whether it's open or closed
  • Settings → Pad → "Notepad" toggle hides the panel and its floating toggle button entirely if you don't use it (on by default)
  • Settings → Pad → "Header" (off by default) adds a title bar with its own close button — turning it on hides the toolbar's close button so there's only ever one; "Match editor background" controls whether Pad's writing surface looks like the outline's canvas or reads as visually distinct chrome — personal preference, no right answer
  • Settings → Pad → "Toolbar buttons" lets you hide individual buttons (Bold, Table, AI Rewrite, etc.) you don't use
  • Toolbar mirrors the Note panel: B I U S, bullet / numbered lists, link (with its own display-text field, separate from the URL), table insert with resizable columns, image paste/insert (click to select, drag the corner handle to resize, double-click to reset), and AI Rewrite (rewrites the selection, or the whole Pad if nothing's selected). No subtree-summarise here — there's no subtree at the document level
  • "Clear pad" permanently removes all content for the current document, after a confirmation prompt — unlike the per-node Note's clear, this isn't on the outline's Ctrl/Cmd+Z undo stack, since Pad content isn't part of that snapshot

Feature activation

  • The button at the top of the Settings panel opens the Features popover — a set of on/off switches for the bigger optional capabilities. Turn off anything you don't use to keep the UI uncluttered; the button turns accent-coloured when any feature is off as a reminder
  • Note Panel — per-node rich-text notes. Off: the note toolbar button, right-click action, note dot on nodes, and the note panel itself are all hidden. Existing note content is preserved — turning it back on restores everything
  • Pad Panel — a docked whole-document notepad, separate from per-node notes. Off: the floating Pad toggle button (bottom-right of the editor) and the Settings → Pad section are hidden. The Pad's content is preserved
  • AI Capabilities — the Rewrite, Expand, Summarise, Generate outline, Suggest icon, and Suggest tags features. Off: the AI toolbar group, all matching right-click items, and the Settings → AI Rewrite section are hidden. Requires a provider API key (Settings → AI Rewrite) before anything will run
  • Decision Log — structured decision records inserted under a node. Off: the toolbar insert button, right-click action, Decision Log Excel export, and the Settings → Decision Log section are hidden. Existing logs already in a document still render normally
  • Tagging — #tag chips on nodes. Off: the tag toolbar button, right-click action, and tag chips on all nodes are hidden. Tags are still stored in the document data; turning the feature back on restores them
  • @ Mentions & Backlinks — type @ while editing to link to another node by name; the Note panel shows incoming backlinks. Off: the @ suggestion dropdown is suppressed and the Backlinks section in the note panel is hidden. Existing [[mentions]] already in node text still render as clickable links regardless of this setting
  • Focus — zoom into a single branch so only it and its descendants are visible. Off: the zoom-in toolbar button and right-click action are hidden, and Ctrl/Cmd + . has no effect. Any active focus session exits immediately when the feature is turned off
  • Turning a feature off hides every entry point for it (toolbar, right-click, Settings sub-section, Help sub-section). The underlying data is never deleted — toggling a feature is non-destructive in both directions. Feature states are saved alongside your other preferences and restored on the next launch
